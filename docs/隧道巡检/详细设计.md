## 1. 系统概述

### 1.1 项目背景

隧道、涵洞与井下巷道是典型的 **GNSS 拒止、弱光、高湿、狭长** 环境，人工巡检效率低、安全风险高。本项目研制一台可自主行进的巡检机器人，在无卫星定位条件下完成 **建图—定位—导航—感知—缺陷检测** 的闭环作业，并对隧道结构健康隐患（裂缝、渗水、衬砌剥落与变形）进行离线检测与三维定位，输出可追溯的缺陷报告。

软件以开源 **GLIM**（GPU 加速的 LiDAR-Inertial 建图框架）为底座，复用其滑动窗口因子图优化、子地图/全局地图、离线交互式回环编辑能力，并在其全局回调插槽机制上扩展先验地图定位、可通行区域感知、安全监控与缺陷检测等子系统。

### 1.2 总体目标与验收指标对照

下表将 README 中列出的项目目标拆解为可验证的工程指标，并标注由哪个子系统负责（详见第 4 章）。子系统章节按需求顺序编排：基础链路（数据管理 → 建图 → 定位 → 感知 → 安全 → 恶劣环境）在前，加分项（G7 无图自主探索、G8 工程化）列于其后。

| 序号 | 目标 | 关键指标 | 责任子系统 |
|------|------|----------|-----------|
| G1 | 数据管理 | rosbag/点云/图像视频完整采集；相机覆盖壁/顶/底全内壁 + 主动照明保证弱光成像；≥ 5 km 连续巡检续航 | §4.1 数据管理 |
| G2 | 多传感器融合感知 | 在线环境感知，输出可通行区域 | §4.4.1 可通行区域感知 |
| G3 | 多传感器融合导航 | 建图完整无盲区；途经点全局定位误差 ≤ 路径长度的 0.5% 且 ≤ 2 m；异常处置（失联保护、自动回收/返航、避障） | §4.3 定位 + §4.4.2 导航避障 + §4.5 安全 |
| G4 | 离线缺陷检测 | 离线对录像做实例分割缺陷检测；缺陷定位到 3D 彩色地图；生成缺陷报告。缺陷类型：结构裂缝、渗水/积水、衬砌剥落与变形 | §4.4.3 缺陷检测 |
| G5 | 电气与软件安全 | 急停/限速/防护；系统自检/故障降级/安全回收；数据完整性与抗干扰 | §4.5 安全 |
| G6 | 恶劣环境 | 20 cm 水深；湿滑/弱光/狭长通道/高湿工况稳定通行 | §4.6 环境适应（贯穿全系统） |
| G7 | 多传感器融合建图（加分） | **无先验地图** 自主探索建图；输出轨迹、里程计与彩色点云地图 | §4.7 无图自主探索（复用 §4.2 建图） |
| G8 | 工程化与可靠性（加分） | 模块化设计、快速部署、低成本高可靠、防护与维护便捷 | §6 工程化 |

> **建图能力的两个层次**：§4.2 建图是 **基础能力**——在已知/遥控引导下对隧道实时建图，产出先验地图（供 §4.3 定位）与彩色点云地图（供 §4.4.3 缺陷检测）；G7 加分项的核心是 **无先验地图的自主探索**（机器人自主决策探索路径并边走边建图），独立设计于 §4.7。

### 1.3 设计原则

1. **复用优先**：最大化复用 GLIM 既有的里程计/建图/回环/离线编辑能力，自研部分集中在隧道场景特化的定位、感知、安全与缺陷检测。
2. **退化优先**：隧道是几何退化（长直、纹理缺失）的重灾区，所有定位/导航环节必须显式建模退化并提供兜底。
3. **在线/离线分离**：实时链路（里程计、定位、导航、安全）追求低时延与确定性；重计算链路（缺陷实例分割、彩色地图重建）放到离线后处理。
4. **安全可降级**：任一传感器或模块失效时，系统能降级到更保守的安全状态（限速 → 停车 → 返航 → 失联保护）。

**关键硬件设计点**

- **相机全内壁覆盖**：多目相机环向布置，视场拼接覆盖隧道壁面、顶拱、底板（G1）。每路相机配同步主动照明 LED，曝光与补光硬触发同步，保证弱光下成像信噪比（G6）。
- **传感器时间同步**：LiDAR / IMU / 相机通过 PTP 或硬件触发统一时基，per-point 时间戳按 GLIM `config_sensors.json` 的 `autoconf_perpoint_times` 配置，保证去畸变精度。
- **防护与续航**：传感器套件按 IP67 及以上防护以适配 20 cm 水深与高湿（G6）；电池容量与功耗预算满足 ≥ 5 km 连续巡检（G1，见 §4.1.4）。

## 3. 复用基础：GLIM 框架能力梳理

本系统以 GLIM 为状态估计底座。下表汇总复用的核心能力及对应代码位置，自研子系统通过 **全局回调插槽** 接入，避免侵入式修改。

| GLIM 能力  | 代码位置  | 本项目用途  |
|-----------|----------|-----------|
| 点云预处理（去畸变/降采样/协方差） | `src/glim/preprocess/` | 统一传感器前处理 |
| LIO 里程计（CT/CPU/GPU 三档） | `src/glim/odometry/` | 实时位姿，GPU 档默认 |
| 子地图构建 | `src/glim/mapping/sub_mapping*` | 局部一致地图 |
| 全局地图 + 位姿图回环 | `src/glim/mapping/global_mapping*` | 全局一致地图（§4.2） |
| 离线交互式回环/编辑 | `src/glim/viewer/offline_viewer.hpp`, `map_editor.hpp` | 离线建图精修（见手动回环指南） |
| 全局回调插槽 | `include/glim/*/callbacks.hpp`, `util/extension_module*.hpp` | 自研子系统注入入口 |

**里程计三档选择**（详见 [summary-odometry.md](../glim/summary-odometry.md)）：

| 档位 | 配准 | 是否用 GPU | 隧道场景适用性 |
|------|------|-----------|---------------|
| CT | CT-GICP, scan-to-model | 否 | 算力受限/备份链路 |
| CPU | GICP/VGICP, scan-to-model | 否 | 中等算力 |
| **GPU** | VGICP, scan-to-keyframes | 是 | **默认**：Jetson Orin 上实时、精度高 |

**关键扩展入口** `on_smoother_update`：iSAM2 优化前触发，回调参数含 `new_factors` / `new_values` 引用，扩展模块可直接注入自定义因子参与本轮求解。先验地图定位（§4.3）即基于此入口注入 Scan-to-Prior-Map 因子。

## 4. 子系统详细设计

### 4.1 数据管理子系统 (G1)

#### 4.1.1 功能描述

- 同步采集并落盘多源数据：rosbag（全话题）、原始/建图点云、多目图像视频。
- 保证相机对隧道全内壁（壁/顶/底）的覆盖，并联动主动照明保证弱光成像质量。
- 提供数据完整性校验、分段录制与磁盘/续航预算管理，支撑 ≥ 5 km 连续巡检（与 G5 数据完整性、G6 弱光成像协同）。

#### 4.1.2 模块组成

```
data_management/
├── recorder_node             # rosbag2 分段录制 + 话题白名单
├── camera_array_driver       # 多目相机驱动 + 曝光/照明同步
├── time_sync                  # PTP / 硬触发时间同步监控
├── integrity_checker          # 帧率/丢帧/时间戳跳变 在线校验
└── storage_manager           # 磁盘配额、分段、断电保护落盘

# ── 暂缓 ──
# illumination_controller      # 主动照明 LED 亮度闭环（按曝光/增益）；当前阶段暂缓，
#                              # LED 先用固定/手动亮度，后续按弱光实测需求再引入闭环
```

#### 4.1.3 关键设计

- **分段录制**：rosbag2 按时长/大小分段（如每 2 min 或 4 GB 一段），降低单文件损坏风险；每段写完即计算校验和（数据完整性，G5）。
- **照明**：主动照明 LED 与曝光帧硬触发同步，降低运动模糊（G6 弱光）。当前阶段 LED 采用 **固定/手动亮度**；按曝光/增益反馈动态调光的闭环模块（`illumination_controller`）**暂缓**，待弱光实测确认必要性后再引入。
- **覆盖自检**：启动自检阶段验证各路相机在线、视场重叠满足壁/顶/底全覆盖，缺路即告警（G1）。
- **续航/容量预算**（见 §4.1.4）：按码率与里程估算磁盘与电量裕度，不足时降码率或提示返航。

#### 4.1.4 续航与存储预算（示例估算）

| 项 | 取值（示例） | 说明 |
|----|-------------|------|
| 行进速度 | 0.5 m/s | 巡检稳态速度 |
| 5 km 用时 | ≈ 2.8 h | 5000 m ÷ 0.5 m/s |
| 多目视频码率 | N 路 × 8 Mbps | H.265 压缩 |
| LiDAR + bag 码率 | ~ 50 MB/s | 按点率估算 |
| 电量预算 | 整机功耗 × 2.8 h × 安全系数 1.3 | 含照明与计算单元 |

> 实测需按最终传感器型号与本体功耗标定本表；该表作为采购与电池/存储选型依据。

### 4.2 多传感器融合建图子系统

> 建图是 **基础能力**：在已知/遥控引导下对隧道实时建图，产出先验地图（供 §4.3 定位匹配）与彩色点云地图（供 §4.4.3 缺陷三维定位）。**无先验地图的自主探索建图**（G7 加分项）在此能力之上增加探索决策，独立设计于 §4.7。

#### 4.2.1 功能描述

对隧道环境实时建图，输出 **轨迹、里程计与彩色点云地图**，并支持离线回环精修生成全局一致地图。

#### 4.2.2 处理流程

```
LiDAR + IMU ─▶ GLIM 预处理 ─▶ LIO 里程计 ─▶ 子地图 ─▶ 全局地图(位姿图)
                                  │                        │
                                  ▼                        ▼
                            实时轨迹/里程计           在线/离线回环检测
                                                           │
                              多目影像 ──────────┐         ▼
                                                ▼   offline_viewer 手动/自动回环
                                          [彩色化模块]      │
                                          (点云 ⊗ 影像)     ▼
                                                     全局一致彩色点云地图
```

- **里程计与轨迹**：直接由 GLIM GPU 里程计输出 `T_world_imu`，发布 odometry 与 TF（见 §5）。
- **全局一致性**：GLIM `global_mapping` 维护位姿图，回环约束消除累积漂移；隧道长直段回环稀少，依赖里程计退化抑制（§4.3.4）+ 进出口/分叉处的可识别回环。
- **彩色化**：将多目影像按外参 `T_lidar_camera` 投影到点云，为每点赋色；离线阶段用精修后的轨迹重投影，得到高质量彩色地图（同时供 §4.4.3 缺陷 3D 定位）。
- **离线精修**：复用 GLIM `offline_viewer`（参见 [手动回环操作指南](../glim/offline_viewer_手动回环操作指南.md)），人工修正建图失败点。

#### 4.2.3 输出物

| 输出 | 格式 | 用途 |
|------|------|------|
| 轨迹 | TUM / CSV | 评测、缺陷定位基准 |
| 里程计 | `nav_msgs/Odometry` | 实时导航 |
| 彩色点云地图 | PCD (XYZRGB) / PLY | 缺陷三维定位底图、成果交付 |
| GLIM SimpleMap | GLIM 序列化格式 | 二次编辑、定位先验地图来源 |

### 4.3 先验地图定位子系统 (G3 定位)

> 本子系统的因子级详细设计沿用 [glim_localization_proposal.md](../glim/glim_localization_proposal.md)，此处给出与隧道场景的衔接与指标。完整接口（`PriorMapManager` / `ScanToPriorMapFactor` / `InitialLocalizer` / `LocalizationExtension`）见该方案文档第 4 章。

#### 4.3.1 功能描述

在 §4.2 建好的先验地图上做全局定位，使途经点位姿误差满足 **≤ 路径长度的 0.5% 且 ≤ 2 m**（G3）。核心是把 Scan-to-Prior-Map（S2M）约束、Scan-to-Scan（S2S）约束、IMU 约束在 GLIM 滑动窗口因子图中统一优化，地图缺失时平滑退化为纯 LIO。

#### 4.3.2 因子图结构

```
  X(k-1) ══[IMU]══ X(k)          ══ IMU 预积分因子
    │               │            ── Scan-to-Scan（GLIM 里程计原生）
  [S2S]───────────[S2S]          ┄┄ Scan-to-Prior-Map（自适应权重）
    │               │
  [S2M]┄┄┄┄┄┄┄┄┄┄┄[S2M]
    │               │
  Prior Map ◀───────┘
```

通过 `on_smoother_update` 注入 S2M 因子，与里程计原生因子同窗优化（接入点见 §3）。

#### 4.3.3 自适应权重与状态机

- **重叠率自适应**：overlap < 5% 不加 S2M（纯 LIO）；5%–30% 线性降权；≥ 30% 全权重。
- **状态机**：`UNINITIALIZED → INITIALIZING → TRACKING → LOST`，LOST 触发重定位。
- **初始定位**：支持 FPFH+RANSAC 全局配准、人工指定（RViz 2D Pose Estimate）、已知出入口先验三种方式。

#### 4.3.4 隧道退化处理（关键）

隧道长直段沿轴向几何约束缺失（S2M Hessian 在轴向特征值趋零），是定位漂移主因。处理：

1. **退化检测**：对 S2M Hessian 做特征值分解，识别退化方向（复用 GLIM `uncertainty_estimator` 思路，逆条件数 < 0.1 判退化）。
2. **方向性降权**：仅在退化方向放大噪声/降权，保留非退化方向（横向、竖向）的强约束，避免整体丢弃地图信息。
3. **多源兜底**：退化方向交由 IMU 预积分与（可选）足端里程计支撑，抑制轴向漂移。
4. **回环锚定**：在隧道分叉、立架、出入口等可识别处强制回环，周期性消除轴向累积误差。

#### 4.3.5 定位精度指标核算

| 指标 | 要求 | 达成手段 |
|------|------|----------|
| 全局定位误差 | ≤ 0.5% × 路径长度 且 ≤ 2 m | S2M 全局约束 + 退化抑制 + 回环锚定 |
| 输出频率 | ≥ 里程计帧率 | 复用 GLIM 输出链路 |
| 边界平滑 | 进出地图无跳变 | 重叠率自适应权重 |

### 4.4 多传感器融合感知子系统 (G2 / G3 导航 / G4)

> 感知层统一承担机器人对环境的理解，按需求拆为三部分：**可通行区域感知**（在线，G2，§4.4.1）、**导航与避障**（在线，G3 导航，§4.4.2）、**离线缺陷检测**（离线，G4，§4.4.3）。三者共享 GLIM 输出的位姿、点云与多目影像——前两者服务实时行进决策，后者服务离线成果交付。

---

#### 4.4.1 可通行区域感知 (G2)

**功能描述**：在线融合 LiDAR、深度相机与本体姿态，输出 **可通行区域**，供导航与避障使用。

**处理流程**

```
去畸变点云 ─┬─▶ 地面/坡度分割 ──┐
            │                    ├─▶ 2.5D 可通行栅格 (cost map)
深度避障相机 ┴─▶ 近场障碍补充 ──┘            │
本体姿态/足端状态 ─▶ 可通行性判据(坡度/台阶/水深) ─┘
            │
            ▼
        OctoMap / GridMap ──▶ 导航层 (§4.4.2)
```

**关键设计**

- **可通行性判据**：综合坡度、台阶高度、悬空、积水深度（联动 §4.6 水深检测）判定栅格可通行/代价。
- **狭长通道适配**：栅格分辨率随通道宽度自适应细化，保证狭窄处（G6）不漏障。
- **动态物体**：结合点云时序差分剔除动态干扰，避免误判可通行区域；建图侧可用对象分割移除（GLIM 演示能力）。
- **退化协同**：当感知判定前方为长直无特征段时，向定位子系统（§4.3.4）发退化预警。

**输出**

| 输出 | 消息类型 | 频率 |
|------|----------|------|
| 可通行栅格 | `nav_msgs/OccupancyGrid` / `grid_map_msgs` | 5–10 Hz |
| 障碍点云 | `sensor_msgs/PointCloud2` | 与里程计同 |
| 三维占据 | OctoMap | 1–5 Hz |

---

#### 4.4.2 导航与避障 (G3 导航)

**功能描述**：在可通行栅格上规划巡检路径并实时避障，输出运动指令给宇树 G2 运动控制器；保证 **建图完整无盲区**（覆盖式路径）与异常处置（与 §4.5 安全协同）。

**模块组成**

```
navigation/
├── global_planner       # 沿隧道中轴/覆盖式巡检路径
├── local_planner        # 局部避障 (DWA / TEB 类)
├── coverage_planner     # 无盲区覆盖（保证相机全壁面扫掠）
├── recovery_behaviors   # 卡死/避障失败的恢复行为
└── cmd_arbiter          # 指令仲裁（安全监控可抢占）
```

**关键设计**

- **无盲区覆盖**：覆盖规划保证机器人行进时多目相机扫掠到全部壁/顶/底面（支撑 G1 覆盖与 G4 缺陷不漏检）。
- **失联保护**：与地面站通信中断时本地自治继续巡检并缓存数据；超时进入 **自动返航**（沿已走轨迹回退）。
- **自动回收/返航**：电量/存储不足、严重故障、任务完成时触发返航，沿先验地图/历史轨迹回到起点（G3）。
- **避障**：local_planner 实时绕障；不可绕行时停车并上报。
- **指令仲裁**：`cmd_arbiter` 接受安全监控（§4.5）的最高优先级抢占（急停/限速）。无先验地图时，目标点由自主探索模块（§4.7）下发。

---

#### 4.4.3 离线缺陷检测 (G4)

**功能描述**：离线对采集的多目录像做 **实例分割缺陷检测**，将缺陷 **反投影定位到 3D 彩色地图**，生成结构化 **缺陷报告**。缺陷类型：结构裂缝、渗水/积水、衬砌剥落与变形。

**处理流程**

```
录制视频/影像 ─▶ 抽帧 + 去模糊/增强(弱光) ─▶ 实例分割模型
                                                  │
                                                  ▼
                                        缺陷掩膜 + 类别 + 置信度
                                                  │
   彩色点云地图(§4.2) + 精修轨迹 ──────────────────┤
   相机外参 T_lidar_camera ─────────────────────── ┤
                                                  ▼
                                   缺陷反投影 → 缺陷 3D 位置/范围
                                                  │
                                                  ▼
                                   缺陷台账 (位置/类型/尺寸/严重度)
                                                  │
                                                  ▼
                                   缺陷报告 (含三维定位图 + 影像证据)
```

**关键设计**

- **实例分割**：按缺陷类别训练实例分割模型（裂缝/渗水积水/剥落/变形各为一类或多类），输出像素级掩膜、类别与置信度。
- **缺陷 3D 定位**：用检测时刻相机位姿（来自精修轨迹）与外参，将掩膜反投影到彩色点云地图，确定缺陷在隧道里程上的三维位置与范围。
- **量化**：裂缝估长宽、剥落/变形估面积与深度（结合点云几何），给出严重度分级。
- **去重与聚合**：同一缺陷在多帧/多相机中出现时按 3D 位置聚类去重，避免重复计数。
- **报告**：生成含缺陷清单、三维定位标注图、原始影像证据、里程桩号的报告（PDF/HTML），可追溯到原始 bag 帧。

**与建图的衔接**：缺陷定位精度直接依赖彩色地图与轨迹精度，因此 **离线先运行 GLIM offline_viewer 回环精修**（§4.2 建图）再做缺陷反投影。

### 4.5 电气与软件安全子系统 (G5)

#### 4.5.1 功能描述

提供急停/限速/防护、系统自检/故障降级/安全回收、数据完整性与抗干扰能力，是所有运动与作业的安全闸门。

#### 4.5.2 安全机制矩阵

| 类别 | 机制 | 触发/实现 |
|------|------|-----------|
| 急停 | 硬件急停回路 + 软件急停话题 | 物理按钮断电；`/emergency_stop` 抢占 cmd |
| 限速 | 分级限速 | 弱光/狭窄/湿滑/退化场景自动降速 |
| 自检 | 上电自检 (BIT) | 传感器在线、外参、时间同步、磁盘/电量 |
| 故障降级 | 传感器/模块降级 | 失某传感器→切备份链路（GPU→CPU 里程计、视觉→纯 LIO） |
| 安全回收 | 自动返航 | 电量/存储/通信/故障阈值触发（§4.4.2） |
| 数据完整性 | 校验和 + 分段 + 断电保护 | §4.1.3 |
| 抗干扰 | EMC 防护 + 通信冗余 | 屏蔽/接地；Wi-Fi/5G/系绳冗余 + 本地自治 |

#### 4.5.3 安全监控节点设计

```
safety_monitor (生命周期节点，最高优先级)
├── 订阅: 诊断(/diagnostics)、电量、通信心跳、定位状态、感知障碍
├── 状态机: NORMAL → DEGRADED → SAFE_STOP → RETURN_HOME → E_STOP
├── 输出: cmd 抢占(限速/停车)、声光告警、返航指令
└── 看门狗: 各关键节点心跳超时 → 降级/停车
```

- **看门狗与心跳**：基于 ROS2 `diagnostics`，关键节点（里程计、定位、感知、运控）周期上报；超时即降级。
- **故障降级链**：GPU 里程计失败 → CPU/CT 里程计；定位 LOST → 纯 LIO + 限速 + 重定位；感知失效 → 停车。
- **失联保护**：通信心跳丢失 → 本地自治 + 数据缓存 → 超时返航（呼应 G3）。

恶劣环境适应贯穿硬件与各子系统，集中归纳如下：

### 4.7 无图自主探索导航 (G7，加分项)

> G7 加分项的核心是 **无先验地图条件下的自主探索建图**：机器人在未知隧道中自主决策探索路径，边走边建图，输出轨迹、里程计与彩色点云地图。本子系统不重复造轮子——建图复用 §4.2、可通行感知复用 §4.4.1、局部避障复用 §4.4.2、状态估计复用 GLIM LIO，仅新增 **探索决策层**。

#### 4.7.1 功能描述

- 在 **无先验地图、无人工引导** 条件下，自主选择前进方向并完成隧道全覆盖建图。
- 处理隧道拓扑（直行、分叉、尽头）下的探索决策与回溯。
- 实时输出轨迹/里程计/彩色点云地图，探索结束后可直接转入 §4.3 先验地图定位模式。

#### 4.7.2 系统结构

```
            ┌──────────────────────────────────────────────┐
            │            探索决策层 (本子系统新增)           │
            │  前沿检测 → 信息增益评估 → 目标选择 → 回溯管理 │
            └───────────────┬──────────────────────────────┘
                            │ 探索目标点
            ┌───────────────▼──────────────────────────────┐
   复用 ───▶│ §4.4.2 导航与避障  →  §4.4.1 可通行区域感知    │
            │ §4.2 在线建图(LIO+子图+全局图+在线回环)        │
            │ §4.5 安全监控(全程兜底)                        │
            └──────────────────────────────────────────────┘
                            │
                            ▼
              轨迹 / 里程计 / 彩色点云地图 + 探索覆盖率
```

#### 4.7.3 探索策略

- **前沿检测（frontier-based）**：在可通行栅格（§4.4.1）上提取“已知可通行/未知”边界作为候选前沿。
- **信息增益评估**：对候选前沿按预期新增覆盖、到达代价、与隧道主轴的一致性打分，优选信息增益高且代价低者。
- **隧道拓扑决策**：识别直行/分叉/尽头——分叉处记录未探支路入栈，尽头或无前沿时 **回溯** 到最近未探分叉，保证无盲区。
- **在线回环**：探索过程中触发 GLIM 在线回环，抑制长程漂移（隧道长直段回环稀疏，依赖 §4.3.4 退化抑制兜底）。
- **安全约束**：探索目标与速度全程受 §4.5 安全监控约束（限速/急停/电量返航）。

#### 4.7.4 与基础链路的关系

| 能力 | 来源 | 说明 |
|------|------|------|
| 实时建图 | §4.2 | LIO + 子地图/全局地图 + 彩色化 |
| 可通行区域 | §4.4.1 | 前沿检测的输入 |
| 局部避障/运动 | §4.4.2 | 执行探索目标点 |
| 安全兜底 | §4.5 | 探索全程受安全监控抢占 |
| 退化抑制 | §4.3.4 | 长直段漂移控制 |

#### 4.7.5 输出与衔接

输出与 §4.2.3 一致（轨迹/里程计/彩色点云地图/SimpleMap），额外发布探索覆盖率与前沿可视化。探索所得地图经 §4.2 离线精修后，即可作为 §4.3 定位的先验地图，形成 **“一次自主探索建图 → 多次先验地图巡检”** 的闭环。

## 6. 工程化与可靠性 (G8，加分项)

### 6.1 模块化设计

- **GLIM 扩展模块机制**：自研子系统尽量实现为 `ExtensionModule` 动态库，经 `config_ros.json` 的 `extension_modules` 加载，免改 GLIM 主体（参见 §3 与 `util/extension_module*.hpp`）。
- **ROS2 包划分**：按子系统分包（data_management / mapping / localization / perception / navigation / safety / exploration / defect_offline），松耦合、可独立测试与替换。
- **生命周期节点**：关键节点用 ROS2 lifecycle，支持有序启动/降级/恢复。

### 6.2 快速部署

- **容器化**：复用仓库 `docker/` 提供的镜像，固化 CUDA/GTSAM/gtsam_points 依赖，一键部署到 Jetson Orin（Jetpack 6.1）与 x86 工作站。
- **配置外置**：所有参数集中在 `config/*.json` 与各子系统 yaml，现场标定无需重编译。
- **Launch 编排**：分 `bringup`（实车实时）、`explore`（无图自主探索）与 `offline`（离线后处理）三套 launch。

### 6.3 低成本高可靠

- 复用开源 GLIM/GTSAM 栈，零授权成本（对比闭源 GLIL）。
- 三档里程计 + 多链路降级，单点失效不致命。
- 传感器套件 PTS-100 标准化，便于批量与维护。

### 6.4 防护与维护便捷

- 传感器套件模块化快拆，IP67+ 防护，便于清洁与更换（高湿/积水后维护）。
- 完整诊断（`/diagnostics`）与日志（GLIM `config_logging.json`），故障可定位。
- 数据分段落盘，便于按段回放与问题复现。

| 无图自主探索（加分） | 未知隧道自主探索建图 | 无盲区覆盖、自主决策分叉、输出完整地图 |

### 7.2 集成与场景测试

| 场景 | 内容 | 通过标准 |
|------|------|----------|
| 长直退化段 | 长隧道轴向定位 | 轴向漂移受控，满足 0.5% 指标 |
| 地图边界进出 | 进出先验地图范围 | 平滑过渡无跳变，纯 LIO 兜底 |
| 积水 20cm | 涉水通行 | 稳定通行，积水正确标记 |
| 弱光成像 | 照明同步成像 + 缺陷检测 | 影像质量满足检测，缺陷检出 |
| 失联 | 通信中断 | 本地自治 + 缓存 + 超时返航 |
| 故障注入 | 关闭某传感器 | 正确降级，不失控 |
| 无图探索分叉 | 含分叉的未知隧道 | 自主探索全部支路，无遗漏 |

### 7.3 性能指标

| 指标 | 目标 | 平台 |
|------|------|------|
| 里程计/定位频率 | ≥ 10 Hz | Jetson Orin GPU 档 |
| GPU 显存 | < 单卡上限，留裕度 | Orin |
| 实时链路时延 | 感知→指令 < 200 ms | Orin |

## 9. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 长直隧道几何退化导致定位漂移 | 超 0.5% 指标 | 退化检测+方向降权+IMU/足端兜底+回环锚定（§4.3.4） |
| 弱光成像质量差 | 缺陷漏检 | 主动照明同步曝光 + 图像增强（§4.1/§4.4.3） |
| 积水/反光干扰 LiDAR/相机 | 感知误判 | 多传感器交叉校验、积水高代价标记（§4.4.1/§4.6） |
| Jetson 算力不足 | 实时性不达标 | GPU 里程计优化、降采样、离线重计算分离 |
| 通信弱网 | 失联失控 | 本地自治 + 数据缓存 + 超时返航（§4.4.2/§4.5） |
| 缺陷 3D 定位精度依赖建图 | 报告定位偏差 | 先离线回环精修再反投影（§4.2 与 §4.4.3 衔接） |
| 无图探索误入死路/盲区 | 覆盖不全 | 前沿回溯 + 分叉入栈管理（§4.7.3） |
| 高湿结露/镜头起雾 | 成像/传感失效 | 密封防潮、镜头加热、自检结露（§4.6） |
